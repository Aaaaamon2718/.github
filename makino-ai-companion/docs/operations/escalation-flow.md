# エスカレーションフロー

## エスカレーション発生条件

### 自動エスカレーション

以下の条件に該当する場合、AIが自動的にエスカレーションを実行する:

1. **確信度スコアが閾値以下** (デフォルト: 0.5未満)
2. **強制エスカレーションカテゴリに該当**:
   - コンプライアンス関連
   - 個別顧客の契約内容
   - 具体的な保険料試算
   - 税務の最終判断

### AIの応答メッセージ

エスカレーション時、ユーザーには以下のメッセージを表示:

```
「この質問については、より正確にお答えするために牧野に確認いたします。
 回答までしばらくお待ちください。」
```

## エスカレーション対応フロー

```
Step 1: AI伴走システムがエスカレーション判定
  │
  ▼
Step 2: ユーザーへのエスカレーションメッセージ表示
  │
  ▼
Step 3: 運用担当へ通知送信（メール/Slack/LINE）
  │
  ▼
Step 4: 運用担当がログ・内容を精査
  │
  ├── 対応可能 → Step 5a: 回答作成 → ユーザーへ返信
  │                        └→ ナレッジベース（knowledge/）に追加（再発防止）
  │
  └── 判断不可 → Step 5b: 牧野氏へエスカレーション
                           └→ 牧野氏が方針決定 → 回答指示
                                    └→ ナレッジベースに追加
```

## 対応SLA（目標応答時間）

| 緊急度 | 定義 | 目標応答時間 |
|--------|------|------------|
| 高 | コンプライアンス関連、クレーム | 4時間以内 |
| 中 | 専門的な判断が必要な質問 | 24時間以内 |
| 低 | 一般的な未対応質問 | 48時間以内 |

## 通知方法

| 方法 | 設定場所 | 備考 |
|------|---------|------|
| メール | config/settings.yaml > escalation.notification | デフォルト |
| Slack | 別途設定 | オプション |
