# Phase 1: ヒアリング・要件定義

## 基本情報

- **期間**: Month 1 - 2（2ヶ月間）
- **目的**: プロジェクトの土台となる要件を明確化し、AI伴走システム設計の基盤を構築する
- **前提**: 牧野氏の人格・哲学および業務フローを深く理解し、技術要件へ落とし込む

## Month 1: 初期ヒアリング

### 1.1 牧野氏への詳細インタビュー（2-3回）

AI伴走システムの人格形成や教育方針の核となる思想を抽出する。

**重点ヒアリング項目:**
- 営業哲学
- 受講生への想い
- 成功・失敗のエピソード

### 1.2 業務面ヒアリング

| 項目 | 内容 | 目的 |
|------|------|------|
| 対応工数把握 | 1日/1週間あたりの質問対応時間 | AI導入による削減効果の試算基準 |
| 質問チャネル整理 | メール、LINE、会員サイト等の特定 | AI設置場所・連携仕様の要件明確化 |
| 回答困難領域 | 過去に回答に窮した質問の傾向 | エスカレーション基準・SLAの定義 |
| 繁忙期・時間帯偏り | 質問流入の時間的分布 | システム負荷設計への反映 |

### 1.3 人格・言い回しヒアリング

| カテゴリ | 収集内容 |
|---------|---------|
| 口癖・頻出フレーズ | 「誰にでもできることを〜」等の決まり文句、文末表現の傾向 |
| 励ましの言葉 | 自身の失敗談を交えた励まし方、前を向かせる表現 |
| 叱咤・厳しい言葉 | プロとしての基準を示す言葉、言い訳への切り返し |
| 禁止表現（NGワード） | 若者言葉、ネットスラング、他社批判、不確実な断定表現 |

### 1.4 コンテンツ面ヒアリング

- [ ] 質問集中領域の特定（ボリュームゾーン）
- [ ] 既存資産の棚卸し（FAQ、講義資料、営業台本等）
- [ ] 回答優先度の高い領域の特定
- [ ] 権利関係の確認（著作権、肖像権、第三者資料の利用許諾）

## Month 2: 要件定義・設計

### 2.1 要件定義書の作成

機能要件、非機能要件、システム要件を網羅的に文書化する。

### 2.2 AI伴走システム対応範囲の明確化

- **回答対象領域**: AI伴走システムが回答すべき範囲
- **エスカレーション領域**: リスク回避のため有人対応へ誘導する範囲
- **回答禁止領域**: 絶対にAIに答えさせない範囲

### 2.3 人格トレース設計方針

牧野氏の「口調」「思考プロセス」「価値観」をAIに再現するためのプロンプト設計方針を決定。

### 2.4 データ整理方針策定

非構造化データ（動画・音声・書籍等）→ 構造化データへの変換ルールを策定。
