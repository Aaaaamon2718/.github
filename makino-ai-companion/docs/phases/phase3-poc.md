# Phase 3: PoC開発・技術検証

## 基本情報

- **期間**: Month 5 - 6（2ヶ月間）
- **目的**: 限定範囲でAI伴走システムを構築し、回答の正確性・人格トレース再現性・運用プロセスを評価
- **前提**: Phase 2の成果物（ナレッジベース、人格定義書完成版、言い回し辞書）が整備済み

## Month 5: PoC開発

### 5.1 Dify基盤構築

ノーコードプラットフォーム「Dify」を採用し、迅速かつ柔軟な修正が可能なAI伴走システム基盤を構築する。

### 5.2 ナレッジベースのインポート

Phase 2で整理・構造化したデータをDifyのナレッジベースに取り込み、検索可能なベクトルデータへ変換する。

### 5.3 プロンプト設計（人格トレース）

牧野氏の語彙、口癖、思考プロセス、価値観を反映したシステムプロンプトを設計する。

### 5.4 RAG実装

```
ユーザー質問 → ナレッジベース検索 → 関連情報抽出 → LLM生成（人格プロンプト適用）→ 回答
```

### 5.5 PoC対象範囲

- **公開場所**: 牧野生保塾会員サイト内の限定領域
- **対象質問**: 頻出質問 TOP 30
- **人格トレース**: 基本実装レベル

### 5.6 技術構成図

```
[ユーザー] → 質問 → [Difyエージェント] → RAG検索 → [ナレッジベース]
                          │                              │
                     人格プロンプト                    関連情報
                          │                              │
                          └──── LLM生成 ← ──────────────┘
                               │
                          回答表示 → [ユーザー]
                               │
                          Webhook送信
                               │
                               ▼
                     [Google Apps Script] → [ログ蓄積シート]
                               │
                          定期レビュー → 改善 → [継続的学習サイクル]
```

## Month 6: PoC検証

### 6.1 牧野氏による評価

- [ ] 回答内容の専門的正確性
- [ ] 言い回し・語彙選択の自然さ
- [ ] 「自分が言いそうか」の主観的判定

### 6.2 テストユーザーによる評価

- [ ] 選抜した受講生5〜10名による試用
- [ ] 実際の悩み・疑問に基づいた質問投入
- [ ] 満足度アンケートの実施と回収

### 6.3 評価指標（KPI）

| 指標 | 目標値 | 測定方法 | 改善アクション |
|------|--------|---------|-------------|
| 回答正確性 | 80%以上 | テストユーザーFB・サンプリング | ナレッジ誤情報修正、参照Doc追加 |
| 人格再現度 | 牧野氏承認 | 本人主観レビュー | プロンプト言い回し微調整 |
| 回答不能率 | 20%以下 | ログ自動集計 | 未回答カテゴリ特定・ナレッジ投入 |
