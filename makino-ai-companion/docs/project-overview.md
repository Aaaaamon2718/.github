# プロジェクト全体像

## プロジェクト概要

**プロジェクト名**: 牧野生保塾 AI伴走システム開発プロジェクト
**期間**: 12ヶ月 (Month 1 - 12)
**目的**: 生命保険営業の生産性向上と知見継承を実現する AI 伴走システムの構築

## 対象者

生命保険業界の経営者および運用責任者を対象とし、意思決定に必要な情報を網羅的に提供する。

## 全体スケジュール

```
Month:  M1   M2   M3   M4   M5   M6   M7   M8   M9   M10  M11  M12
        ├────┤                                                        Phase 1: 要件定義・設計
             ├─────────┤                                              Phase 2: データ収集・整理
                       ├─────────┤                                    Phase 3: PoC開発・検証
                                 ├─────────┤                          Phase 4: MVP開発 (Pattern 1)
                                           ├─────────┤                Phase 5: パターン展開 (2-4)
                                                     ├─────────┤      Phase 6: 本番運用・保守
```

## 成果物（エージェント）パターン一覧

| Pattern | エージェント名称 | 対象ユーザー | 主な機能 |
|---------|----------------|------------|---------|
| Pattern 1 | 牧野生保塾 質問対応エージェント | 月額会員 | 汎用Q&A、24時間即時回答、決算書分析、法人保険、ドクターマーケット全般 |
| Pattern 2 | ドクターマーケット特化エージェント | 開業医攻略研修 受講生 | 医療法人アプローチ、医療法・税制知識、面談ロールプレイ |
| Pattern 3 | 法人保険特化エージェント | 法人財務スペシャリスト研修 受講生 | 決算書分析、財務改善提案、退職金設計、事業承継 |
| Pattern 4 | 人格エージェント | 全受講生・営業担当者 | 精神的サポート、牧野氏の哲学に基づく励まし、モチベーション維持 |

## 技術構成

- **基盤**: Dify Platform（ノーコードAIエージェント構築）
- **AI**: LLM (GPT-4等) + RAG (Retrieval-Augmented Generation)
- **データ蓄積**: Google Sheets + Google Apps Script
- **連携**: Webhook API
- **文字起こし**: AI文字起こしサービス

## リスク一覧

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 牧野氏の協力時間確保困難 | 高 | インタビュー日程を半年先まで事前確保 |
| 過去データの品質不足 | 中 | 追加インタビュー収録で補完 |
| 人格トレースの精度不足 | 高 | 2週間サイクルのフィードバックループ |
| 受講生の期待値ギャップ | 中 | β版として限定公開、「共に育てるAI」の位置づけ |
| Dify等のPF仕様変更 | 低 | 定期的な最新情報キャッチアップ |
