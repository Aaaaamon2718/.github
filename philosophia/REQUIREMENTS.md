# Philosophia — 要件定義書

> Amonの哲学的思考を体系化し、一冊の哲学書へと統合するためのナレッジリポジトリ

---

## 1. プロジェクト概要

### 1.1 目的

自分自身の哲学的探求（インプット・アウトプット双方）を一箇所に集約し、最終的に**一冊の体系的な哲学書**として統合・編纂していくための「知的倉庫＋ワークスペース」をGitHub上に構築する。

### 1.2 運用思想

- **Claude Code駆動**: リポジトリの構造化・素材整理・統合作業をClaude Codeで実行する
- **漸進的構築**: 日々の素材投入 → テーマ別整理 → 章立て統合、というフローで段階的に哲学書を育てる
- **原本保持**: スクショ等の原本は常に保持し、OCRテキストは派生物として管理する

---

## 2. 格納するコンテンツの分類

### 2.1 インプット（他者の思想）

| 種別 | 形式 | 処理 |
|------|------|------|
| 書籍スクリーンショット | 画像 (PNG/JPG) | OCRでテキスト抽出 → 引用として構造化管理。原本画像も保持 |
| 書籍メモ・要約 | Markdown | 書籍ごとに整理、引用元情報（著者・書名・ページ）を付与 |
| 外部記事・参考資料 | Markdown / URL | 出典情報付きで保存 |

### 2.2 アウトプット（自分の思想）

| 種別 | 説明 | 想定頻度 |
|------|------|----------|
| 断片メモ (Fragments) | 日々の哲学的な気づき・着想・問い | 高頻度（日次〜） |
| 対話記録 (Dialogues) | Claudeとの哲学的対話から生まれた思考の記録 | 中頻度 |
| エッセイ (Essays) | テーマごとにまとめた考察文 | 低頻度（熟成後） |

### 2.3 統合成果物（哲学書）

- 上記インプット・アウトプットを素材に、テーマ別の**章（Chapter）**として編纂
- 哲学書全体の構成（目次・序論・各章・結論）を管理

---

## 3. テーマ構造

```
themes/
├── love/            # 愛の哲学
├── stoicism/        # ストア哲学
├── fromm/           # エーリッヒ・フロム
├── creativity/      # 創造性の哲学
└── _emerging/       # 未分類・萌芽的テーマ
```

テーマは固定ではなく、素材が蓄積される中で追加・統合・再編される前提。

---

## 4. 運用フロー

### 4.1 日常の素材投入

1. 書籍スクショを `input/books/{book}/screenshots/` に投入
2. Claude Codeで OCR処理 → `ocr/` にテキスト生成
3. 気づきがあれば `output/fragments/` に断片メモを作成
4. Claudeとの対話が実りあれば `output/dialogues/` に記録

### 4.2 定期的な統合作業

1. `themes/` の索引を更新
2. 十分な素材が溜まったテーマについてエッセイ化 → `output/essays/`
3. エッセイを素材に `synthesis/chapters/` のドラフトを更新
4. `synthesis/outline.md` で全体構成を見直し
